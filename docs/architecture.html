Mission

Prove that the CEO, Claude, ChatGPT, and Gemini can share one durable local room where state lives outside every vendor surface.

Core Process

CEO browser or wt-say
        |
        v
wt-roomd HTTP/SSE server
        |
        v
data/phase0-room.jsonl
        |
        v
wt-agent --agent claude|chatgpt|gemini

Why Native C

The Phase 0 core is the durable room fabric for a limited-resource GNU/Linux host. It avoids Docker, Node, Python, PHP, and framework runtimes so the running system stays small, inspectable, and cheap.

Build And Run

make clean
make
cp config/woventeam-phase0.conf.example config/woventeam-phase0.conf
./build/wt-roomd --config config/woventeam-phase0.conf
./build/wt-agent --agent claude --loop
./build/wt-agent --agent chatgpt --loop
./build/wt-agent --agent gemini --loop
./build/wt-say ceo all "Claude, ChatGPT, Gemini: each confirm."
./build/wt-tail --limit 20

Access

The daemon binds to 0.0.0.0:8787 by default so a headless intranet host can serve the browser UI at http://HOSTNAME_OR_IP:8787/.

Future Extension Points

Redis, Slack, systemd services, vendor CLI adapters, indexing, authentication, TLS, and multi-room dashboards are intentionally outside this spike and can be added behind the same room-store boundary.
