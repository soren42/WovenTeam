WovenTeam Threat Model

Security boundary for the Phase 3 Sprint 4 remote execution substrate: single-tenant intranet, daemon-owned policy, authenticated remote agents, append-only audit, and explicit hardening gates before cross-network operation.

Updated 2026-05-21 · Scope: wt-roomd, remote agent bearer auth, host capabilities, task leases, reclaim, and operator mutation gates
Scope Boundaries Controls Out of Scope Hardening

Scope

In Scope

Daemon Policy Authority

WovenTeam is currently modeled as a single-tenant operator workbench on a trusted intranet. wt-roomd remains the policy and audit origin.

In Scope

Remote Agent Substrate

Remote agents authenticate, advertise capabilities, claim leased tasks, and append task lifecycle events through the daemon; they do not become policy authorities.

Sprint 4 update: bearer token and token-id generation now uses /dev/urandom entropy. Token material is still present in the JSONL ledger, so hash-only storage remains a hardening follow-up.

Trust Boundaries

Local Operator Origin

Loopback requests are treated as operator-origin for the current TCP daemon. A future Unix-domain socket should replace this compatibility shortcut.

Remote Agent Origin

Bearer token plus IP allow-list is required for /api/host-capabilities, /api/remote-claim, and /api/remote-task-event.

Operator Bearer Origin

Mutating operator endpoints require either local origin or a bearer token with role operator.

Audit Origin

Token, host capability, lease, reclaim, cancel, autonomy, artifact, and deliverable actions are append-only JSONL records.

Current Controls

Active

Bearer Tokens

Tokens carry tokenId, role, subject, issuedAtUnixMs, and ttlSeconds. The token body contains 128 bits of OS CSPRNG entropy.

Active

Revocation

/api/auth-token/revoke appends a revoke event; validation rejects the next use with token_revoked.

Active

IP Allow-list

remoteAllowedIps constrains bearer-authenticated remote calls before token validation.

Active

Host Capabilities

Capabilities are recorded as woventeam.host_event.v0.1 records and exposed through /api/hosts.

Active

Host Pinning

executionHost pins a task to one advertised host. Capability mismatches reject with capability_unmet.

Active

Lease Reclaim

Lease expiry and reclaim use the existing append-only reclaim path, so network partitions surface as stuck leases and recover without a separate remote state machine.

Out Of Scope

Deferred

Tenant Isolation

Multi-tenant isolation is not part of the current single-tenant workbench boundary.

Deferred

Internet Exposure

Internet-exposed daemon operation remains out of scope without mTLS or a reverse proxy.

Deferred

Secret Storage Hardening

Token material is still stored in the JSONL ledger. The follow-up is to store token hashes only.

Deferred

Remote Filesystem Sandbox

Filesystem sandboxing beyond the adapter/tool policy already recorded on tasks is not yet implemented.

Required Hardening Before Cross-Network Use

  • Put wt-roomd behind mTLS or add native TLS client certificate validation.
  • Move token secrets out of the audit ledger, storing only hashed token material.
  • Replace loopback TCP operator bypass with a Unix-domain socket or explicit local operator session.
  • Add per-role token issuance policy so remote agents cannot request operator-class tokens through local bootstrap paths.

The current acceptance test remains loopback-based on hosts that reject unprivileged network namespaces; a privileged run should repeat the same assertions across container or netns boundaries.
