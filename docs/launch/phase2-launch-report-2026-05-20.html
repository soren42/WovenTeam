Live Adapter Rehearsal

bin/wt-rehearse-live drives one bounded real-CLI task end-to-end against an installed adapter. The script is operator-gated: default mode is dry-run and --yes is required to actually post the task. The script writes a Markdown rehearsal log and a JSON audit dump under docs/launch/phase2-rehearsal-<UTC date>.md.

# Dry run: prints adapter readiness + policy knobs + planned task.
./bin/wt-rehearse-live --agent claude --max-tokens 4000

# Execute (operator-approved tokens, real CLI is invoked).
./bin/wt-rehearse-live --yes --agent claude --max-tokens 4000 \
  --initiative init_rehearse_$(date -u +%Y%m%d)

Safety: the script refuses to run if the chosen adapter is not enabled + launchable, and refuses without --yes. It exits non-zero on any of: unreachable daemon, unhealthy projection, missing adapter readiness, or task creation failure.

Deferred Beyond Phase 2

  • Topology view, command palette, audit query view. Each needs its own backend query endpoint; the disabled-control pattern in the console keeps them visible-but-inert until they land.
  • Audit pagination. The 1 MiB single-buffer audit fits today's initiatives but should grow into a windowed model post-launch.
  • Lease renewal for very long real-CLI runs. 15-minute leases are sufficient for current adapters; a heartbeat-based renewal is the next workload-control enhancement.
  • Actual-usage budget enforcement. Per-model-family budgets today gate on allocated maxTokens; an optional actual-usage variant is a small addition.
  • Bulk reclaim / bulk promotion CLI verbs. Operator workflow data from the first real rehearsal will tell us if these are needed.

No new risks introduced by Sprint 6. The launch rehearsal harness reuses every Phase 2 endpoint that existed at the close of Sprint 5; the only new artifact is the operator-gated wt-rehearse-live script and the additive Phase 2 sections in the runbook + cheat sheet.
