Detailed Technical Status

Ledger Schema

  • New eventType:"artifact" task event with allowlisted artifactState (draft, reviewed, accepted, rejected, superseded), reviewer, reviewNotes, and artifactPath.
  • Append helper wtTaskAppendArtifactEvent() in wt_task_store.c escapes every field and respects fsyncEachMessage.
  • Older ledgers without artifact events project as empty values - the schema is additive.

Projection

  • Five new tasks columns: artifact_state, last_reviewer, last_review_notes, accepted_at_ms, accepted_artifact_path, plus an idx_tasks_artifact index.
  • accepted_at_ms and accepted_artifact_path only latch on accepted events, so the inventory keeps a stable "what was accepted, by whom, when" record even after a later supersede.
  • Task summaries and detail JSON now include the artifact fields; clients can read state without an extra round-trip.
  • New endpoint helper wtTaskProjectionReadInitiativeArtifactsJson() powers GET /api/initiative-artifacts with accepted-first sort.

Endpoints

  • POST /api/task-artifact validates state against the allowlist, runs paths through isSafeArtifactName, appends the artifact event, and broadcasts a task.artifact room message.
  • GET /api/initiative-artifacts?initiativeId=... returns { artifacts: [...], acceptedCount, pendingCount } with reviewer / notes / accepted-path inline.

CLI

  • wt-task artifact promote|reject|review|supersede TASK_ID [--path PATH] [--reviewer NAME] [--notes TEXT] covers the state transitions; CLI verbs are mapped to schema-level states (promote → accepted, review → reviewed, reject → rejected, supersede → superseded).
  • wt-task artifact note TASK_ID --notes TEXT records a reviewed state with operator notes only.
  • wt-task artifact list INIT prints a TSV inventory; wt-task artifact export TASK_ID [--out FILE] streams the accepted (or default result.md) text from the workspace.

Artifact Lifecycle

All transitions are append-only events. The projection holds the latest state in artifact_state with reviewer + notes; accepted_* columns latch on the first accepted event and survive later supersede transitions.

Acceptance is the only state that latches a path - the inventory deliberately tracks "the artifact that was accepted at time T" even if a later supersede changes the live state.
