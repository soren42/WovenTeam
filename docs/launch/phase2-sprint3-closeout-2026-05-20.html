Detailed Technical Status

Ledger Schema Additions

  • woventeam.task_event.v0.1 now supports eventType:"reclaim". The event carries reclaimReason (operator or lease_expired), assignedAgent (previous lease holder), and resets status to queued.
  • Status events emitted by wt-agent for failed attempts include an optional retryCause string. Existing callers without a cause continue to emit the previous event shape unchanged.
  • wtTaskAppendReclaimEvent() and wtTaskAppendStatusEventWithCause() are the new append helpers in wt_task_store. Both are JSON-string-escaped and respect the existing fsyncEachMessage setting.

Projection And API

  • The tasks projection table gained three columns: failure_cause, last_reclaim_reason, reclaim_count. A reclaim event clears the lease columns so a follow-up lease event reads cleanly.
  • GET /api/task-summaries and GET /api/task-detail now include failureCause, lastReclaimReason, and reclaimCount in their JSON output.
  • POST /api/task-reclaim accepts taskId, reason (allowlist: operator, lease_expired), optional message and createdBy. The handler looks up the current lease holder so the recorded event has provenance.
  • A 256-byte to 320-byte buffer fix in wtTaskProjectionReadSummariesJson resolves a latent truncation bug uncovered while wiring the new fields - the prior 128-byte stack number buffer would silently chop the response when ledger key counts grew.

Agent Runtime

  • New wtTaskFindClaimableForAgent() returns the next queued/assigned task or, if none, a task whose status is leased/running but whose lease has expired. Routing/assignment events drive the rightful-owner check.
  • Before recording a new lease, handleAssignedTask calls wtTaskFindActiveLease(). A still-valid foreign lease causes the agent to skip the task. An expired lease is released with a lease_expired reclaim event before the new lease is appended.
  • Each lease event now records attempt = previousAttempt + 1, so attempt counts are coherent across recovery cycles.
  • Codex and CLI artifact adapters classify their own exit codes: timeout (signal/kill timer fired), adapter_unavailable (exec returned 127), exit_nonzero (any other non-zero exit). The classification is emitted on the failed status event.

Operator Interfaces

  • bin/wt-task reclaim TASK_ID [--reason operator|lease_expired] [--message TEXT] posts to the new endpoint.
  • The web console renderAgentControls() now adds a RECLAIM TASK button to any agent card whose stuckTasks > 0. Clicking the button reclaims the currently-selected task.
  • The task detail meta line now surfaces attempt N, cause: X, lease: AGENT, and reclaimed N× (reason) when those fields carry signal.

Operator Quick Reference

CLI

# List agent state and workload counters
wt-task agent list

# Pause or resume an agent (durable; survives restarts)
wt-task agent pause chatgpt --message "Codex CLI maintenance"
wt-task agent resume chatgpt --message "Codex CLI restored"

# Reclaim a stuck task (releases its most recent lease)
wt-task reclaim task_abc123 --reason operator \
  --message "Manual unblock - stuck after restart"

API

POST /api/task-reclaim
Content-Type: application/json

{
  "taskId": "task_abc123",
  "reason": "operator",          // or "lease_expired"
  "message": "Manual unblock",
  "createdBy": "ceo"
}

200 OK
{ "ok": true,
  "taskId": "task_abc123",
  "reason": "operator",
  "previousAgent": "chatgpt",
  "previousAttempt": 1 }

Web Console

The agent control panel under "Agents in flight" gains a RECLAIM TASK button on any card with stuckTasks > 0. Click the stuck task in the task list first to select it, then press RECLAIM TASK. The task detail meta line shows the latest attempt, failureCause, lease holder, and reclaim history so the operator can confirm the action without inspecting raw events.

Follow-ups Into Sprint 4 (Artifact Promotion)

  • Adapter lease renewal remains a future enhancement: long real-CLI runs that approach 15 minutes still need a renewal heartbeat. Sprint 4 work is unaffected, but the topic should land in Sprint 5 policy work alongside per-initiative budget timers.
  • The reclaim button currently targets the selected task. A future "Reclaim all stuck for AGENT" action could iterate task summaries; deferred until artifact promotion lands so the workflow can mark accepted assets as untouchable.
  • The projection still rebuilds the entire SQLite read model on every API hit. With the new columns this remains within budget for Phase 2 scales, but Sprint 6 launch-rehearsal load testing should measure rebuild time on a realistic ledger.

No new risks were introduced by this closeout. The schema additions are additive; older ledgers without reclaim or retry-cause events project as empty values and the UI degrades gracefully.
