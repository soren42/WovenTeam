Detailed Technical Status

Ledger And Agent Runtime

  • wt-agent checks the latest woventeam.agent_control.v0.1 record for its agent before taking work.
  • Paused agents return without claiming, preserving queued work for later processing.
  • Claimed work now gets a woventeam.task_event.v0.1 lease event with status:"leased", assignedAgent, attempt, and leaseExpiresAtUnixMs.
  • The current lease duration is fixed at 15 minutes and is used by projection-level stuck detection.

Projection And API

  • The task projection stores lease owner, lease expiry, leased timestamp, running timestamp, and attempt count.
  • The projection rebuild also derives agent_controls from append-only agent control records.
  • GET /api/agents returns state, control message, active tasks, leased tasks, running tasks, stuck tasks, attempts, and timestamps for claude, chatgpt, and gemini.
  • POST /api/agent-control validates supported agents and pause|resume actions before appending a durable control record.

Operator Interfaces

  • bin/wt-task agent list prints agent state and workload counters as a compact tabular view.
  • bin/wt-task agent pause AGENT --message TEXT and resume use the same HTTP API as the browser.
  • The web console adds agent control cards under the status panel with pause/resume buttons and workload counters.
  • The web task progress model recognizes leased as an intermediate state between queued/assigned and running.

Documentation

  • README.md includes the new agent control commands.
  • docs/api/task-ledger-v0.1.md documents lease events, agent control records, and the new endpoints.
  • docs/ui/phase0-console-scaffold.md documents the console surface for agent controls.
  • docs/launch/phase2-spike3-workload-control-2026-05-19.md records the spike scope, model, validation, and remaining work.

Outstanding Problems And Review Focus

Problem

No Automatic Reclaim

Stuck work is visible, but there is no operator action or daemon job that requeues or cancels expired leases. Reviewers should confirm this is acceptable for the current phase boundary.

Problem

No Lease Renewal

Long-running real adapters can exceed the fixed 15-minute lease window. The next implementation should add heartbeat or renewal events before enabling longer tasks.

Risk

Attempt Semantics Are Basic

Attempts increment on lease projection, but retry causes, terminal failure reasons, and retry limits are not yet enforced centrally.

Risk

Agent Set Is Static

The agent summary endpoint currently returns the three primary agents directly: claude, chatgpt, and gemini. Later phases should derive this from model/agent configuration.

Risk

Policy Is Still Distributed

Vendor/model policy checks exist in pieces. Phase 2 Sprint 5 should centralize policy before broader real-adapter execution.

High-Level Phase 3 Plan

Phase 3 should convert the Phase 2 functional workbench into a dependable collaborative execution environment. The central theme is controlled autonomy: agents should be able to run meaningful bounded work, recover from interruption, surface artifacts for review, and stay inside model, budget, and system-access policy.

Runtime Reliability

Add lease renewal, explicit reclaim/cancel/retry operations, task concurrency limits, crash recovery, and long-run adapter supervision.

Config-Driven Agents

Move static agent lists and route assumptions into validated configuration with live UI visibility and safe reload behavior.

Review Workflows

Make accepted artifacts, review gates, and audit exports the default path from raw model output to trusted project material.

Security And Access

Formalize filesystem, network, shell, and git permissions per tool policy and adapter profile before broadening task scopes.

Operator Experience

Make the web console the primary workbench for initiative state, task routing, agent health, budgets, artifacts, and recovery actions.

Distributed Readiness

Prepare for worker-node execution by separating local daemon assumptions from agent runtime placement, identity, and telemetry.
