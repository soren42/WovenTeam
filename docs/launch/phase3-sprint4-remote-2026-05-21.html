Detailed Technical Status

Remote Agent Surface

  • wt-agent --remote talks to wt-roomd over HTTP using bearer auth.
  • --host binds the worker to an advertised execution host so the daemon can enforce host-pinned tasks.
  • --ssh HOST preserves the operator-run transport hook for later hardened remote launch flows.

Daemon Endpoints

  • POST /api/host-capabilities records woventeam.host_event.v0.1 capability updates.
  • GET /api/hosts returns the roster consumed by operators and the browser console.
  • POST /api/remote-claim grants a lease only when bearer auth, host pinning, and profile requirements pass.
  • POST /api/remote-task-event appends authenticated lifecycle updates from the remote agent.

Routing + Capability Policy

  • executionHost pins task routing to one host.
  • Advertised profile mismatches reject with capability_unmet before the task is accepted.
  • Lease expiry uses the existing reclaim path, avoiding a second remote-only state machine.

Operator Auth Gates

  • Mutating endpoints now require either local operator origin or an operator bearer token.
  • Covered endpoints: task-priority, task-reroute, task-cancel, autonomy-revoke, auth-token, and task-deliverable.
  • Bearer token and token-id generation uses /dev/urandom, keeping existing wt_ and tok_ prefixes.

Security Documentation

  • docs/security/threat-model.html records the single-tenant intranet boundary, remote-agent trust boundary, active controls, and hardening gates.
  • Strong OS CSPRNG token generation is now implemented and removed from out-of-scope.
  • Hash-only token storage remains a required follow-up because token material is still present in JSONL.

Namespace Acceptance

  • The local validation host rejects unprivileged unshare --net and ip netns add, so the checked-in test remains loopback-based.
  • The test now documents exactly what a privileged cross-namespace acceptance run must add.
  • The closeout keeps the real netns/container run as a follow-up instead of masking the environment constraint.

Security Notes

Fixed

CSPRNG Token Material

makeToken() now reads token body and token-id entropy from /dev/urandom. The token body emits 128 bits of entropy while preserving the existing wt_ parsing prefix.

Follow-up

Ledger Secret Storage

Issued token material is still stored in the append-only JSONL ledger so validation can replay from durable history. The next hardening step is hash-only storage.

The threat-model reference is now self-contained HTML at docs/security/threat-model.html.

Follow-ups

  • Operator origin hardening. Replace loopback TCP operator bypass with a Unix-domain socket or explicit operator session.
  • Token storage hardening. Move token material out of JSONL and store hashes only.
  • Cross-network transport. Add native mTLS before supporting remote agents beyond a trusted intranet.
  • Privileged namespace acceptance. Expand the remote integration from loopback-simulated hosts to actual container or network namespaces on a host that supports that setup.
