Phase 3 Sprint 4: Remote Execution Substrate

Closeout date: 2026-05-21 · Branch: codex/phase3-sprint4-remote-execution

Delivered

  • wt-agent --remote URL --bearer TOKEN --host HOST registers host capabilities, claims a daemon lease, and reports running/complete over authenticated HTTP.
  • wt-agent --ssh HOST exposes the operator-launched transport entrypoint.
  • POST /api/auth-token issues role-scoped bearer tokens with issued-at and TTL; POST /api/auth-token/revoke invalidates them with token_revoked.
  • POST /api/host-capabilities records woventeam.host_event.v0.1; GET /api/hosts returns the roster for operators and the web console.
  • executionHost pins task routing to a host, and advertised profile mismatches reject with capability_unmet.
  • Operator-auth gates now wrap task-priority, task-reroute, task-cancel, autonomy-revoke, auth-token, and task-deliverable.
  • docs/security/threat-model.md records the current intranet trust boundary and hardening requirements.

Validation

  • New integration: make test-remote-execution passes.
  • The test runs two remote agents against one daemon, completes host-pinned tasks, simulates lease expiry/reclaim, revokes a token, checks token_revoked, checks capability_unmet, and verifies anonymous remote mutation gets bearer_required.
  • Full regression: make test passes, including the new remote execution test.

Follow-ups
