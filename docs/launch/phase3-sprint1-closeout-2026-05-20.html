Detailed Technical Status

New Schemas

  • woventeam.heartbeat.v0.1 — agent, host, currentTaskId, leaseExpiresAtUnixMs, statusLine. One upsert row per agent in the projection.
  • woventeam.milestone.v0.1 — taskId, milestone (plan|execute|review|escalate|complete), message, createdBy. Append-only projection table.
  • woventeam.kill_event.v0.1 — taskId, reason, createdBy. Polled by the agent wait loop via wtTaskCancelRequested().
  • Existing woventeam.task_event.v0.1 gains a new eventType:"lease_renewal" variant emitted by the wait loop.

Config Knobs

  • heartbeatIntervalSeconds (default 7200) — wt-agent loop heartbeat tick.
  • leaseRenewalIntervalSeconds (default 600) — adapter wait-loop renewal tick.
  • leaseDurationSeconds (default 900) — lease window each renewal extends.
  • cancelPollIntervalSeconds (default 5) — wait-loop ledger poll for kill_events.
  • slackWebhookFile, notificationEscalateKey, notificationStuckKey — webhook lookup.

Endpoints

  • GET /api/status — aggregated payload: initiatives, agents, heartbeats, recent milestones, task summaries, tokens, adapter readiness, policy snapshot, heartbeat interval.
  • POST /api/task-cancel — appends kill_event; wt-agent picks it up in its next poll and tears down the adapter process group.
  • GET /api/initiative-audit — gains ?since=&limit=. Response includes truncated and nextSinceUnixMs for cursoring.

Agent + Daemon

  • New waitForAdapterChild() helper in wt-agent owns the entire wait loop: renewal tick, cancel-poll tick, timeout, SIGTERM-then-SIGKILL of the child's process group. Both adapter paths (Codex, CLI artifact) use it.
  • Adapters now setpgid() their child to be a process-group leader; cancel terminates the whole tree.
  • Cancelled tasks land as status:"cancelled" with retryCause:"operator_cancel". The artifact-promotion path filters these out (cancelled output never promotes).
  • New wt_notify module shells out to curl for webhook POSTs. WT_NOTIFICATION_OVERRIDE_URL bypasses the key lookup for tests.

Console

  • New status-board panel under the status grid: per-agent heartbeats with age + status line, plus the most recent six milestones. Polls /api/status every 7 s.
  • Browser JS only (per project directive); no new build dependency.

CLI

  • wt-task cancel-task TASK_ID [--reason TEXT] [--message TEXT] wraps /api/task-cancel. Distinct from the existing cancel lifecycle verb.

Follow-ups into Sprint 2 (Autonomy)

  • The notification module today fires for policy denials + task cancels. Sprint 2 / 3 should add hooks for escalate milestones + stuck-lease detection alongside the new autonomy_required reason class.
  • /api/status's policy block is currently a snapshot of config knobs; adding "recent policy denial counts in the last hour" would make the status board more glanceable. Defer.
  • The status board polls every 7 s; once the SSE-driven dashboard arrives in Sprint 5, the polling cadence can drop to a heartbeat-only fallback.
  • Audit pagination uses a created-at cursor. For initiatives with bursty timestamps (multiple events in the same ms), the next-page filter can re-emit boundary records; if that becomes an issue, switch to an id-based cursor.

No new risks introduced. Schemas are additive; older ledgers without heartbeats/milestones/kill_events project as empty tables. Cancel semantics use POSIX kill(-pgid, sig) which is portable under _POSIX_C_SOURCE=200809L.
