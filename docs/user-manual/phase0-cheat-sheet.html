End-To-End Example

Kick off a parent task

Use the web composer in Task package mode, or create it from the CLI with `wt-task create`.

Agent claims and reports

The matching `wt-agent@...` loop polls the task ledger, appends `running`, then appends `complete` or `failed`.

Request a child task

Switch the web composer to Manager subtask, enter the parent task ID, and choose a spawnable role. The daemon validates role policy.

Watch the result

The browser task table and transcript update from `GET /api/tasks` and `/events`. CLI users can run `wt-task list` or `wt-task show TASK_ID`.

Web Console Pattern

CreateComposer mode Task package: title + task brief + role, then ARM and EXECUTE.
SubtaskComposer mode Manager subtask: parent task ID + requester role + target role.
ObserveUse the task table for status, parent, dependencies, requested-by role, assigned role, and agent.
DisabledGrey controls are intentionally inert until their backend APIs exist.

Role Spawn Rules

Status Meanings

queuedTask package exists and is waiting for a matching agent.
runningAn agent accepted the task and wrote a `task.status` event.
completeThe agent wrote a final `task.result` event.
blockedThe task cannot proceed. Dependents are marked blocked automatically.
failedAdapter or runner exited unsuccessfully.

One Live Shell Walkthrough

ROOM_URL=http://127.0.0.1:8787

parent_json="$(WT_ROOM_URL="$ROOM_URL" ./bin/wt-task create \
  --title "CEO kickoff from cheat sheet" \
  --body "Run the live Phase 0 parent task path." \
  --role project_manager \
  --agent claude \
  --initiative init_cheat_sheet)"

parent_task="$(jq -r '.taskId' <<<"$parent_json")"

child_json="$(WT_ROOM_URL="$ROOM_URL" ./bin/wt-task request \
  --parent "$parent_task" \
  --by-role project_manager \
  --role backend_dev \
  --agent chatgpt \
  --initiative init_cheat_sheet \
  --title "Worker child from cheat sheet" \
  --body "Let the live agent claim this and report a result.")"

child_task="$(jq -r '.taskId' <<<"$child_json")"

sleep 3
WT_ROOM_URL="$ROOM_URL" ./bin/wt-task show "$parent_task"
WT_ROOM_URL="$ROOM_URL" ./bin/wt-task show "$child_task"
